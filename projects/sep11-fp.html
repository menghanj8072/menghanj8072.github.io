Freedom Project

The "Freedom Project" for SEP11 is all about learning and exploring new tools that will be useful for our project. The purpose of this blog is to show my process from how I learn my tool to the final product. For my freedom project this year in SEP11, I'm making a game that is similar to the game of hungry shark using the javascript game programming library: KaboomJs. This blog will walk through my process of independently studying of KaboomJs in order to make my final product.



Challenge/s

For most of the time, I didn't face any challenges, but when I'm working on my MVP(minimum viable product) I faced one challenge. For my game, the sprites are all in png file. When I'm making the sprites, I left too much blank spaces around the sprite which results a problem. When the player sprite is still far away from the bomb, the game ended. It's that the sprite did collide with the bomb and the game ended, however on the player's screen they will see that the sprite didn't collide with the bomb because of the invisible spaces. To solve this issue, I went to check on the sprites that I made and I crop the image to reduce the blank spaces around the sprite. Now on the player's screen they will see that when the sprite is very close bomb, the game ended.


Takeaway/s

  • KaboomJs is different from the Javascript that we learned in class. In Kaboom, we mainly using arrow functions. Learning how to write arrow functions is one of my takeaways.
  • My second takeaway was that I learned to use location.reload() to reload the page.
